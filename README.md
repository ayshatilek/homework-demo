# DRY, KISS, YAGNI – Java Demo

## Описание
Данный проект демонстрирует применение базовых принципов проектирования:
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple, Stupid)
- YAGNI (You Aren't Gonna Need It)

Код написан на Java и содержит примеры корректной переработки исходных фрагментов.

---

## Используемые принципы

### DRY
- Один параметризованный метод логирования
- Общая конфигурация подключения к базе данных

### KISS
- Минимальная вложенность условий
- Отказ от избыточного LINQ/Stream API
- Проверки вместо исключений

### YAGNI
- Класс User содержит только нужные поля и методы
- Убраны лишние настройки и неиспользуемые функции
- Отчёт генерируется одним методом
